    19|## Overview
    20|
    21|### What is x402?
    22|
    23|**x402** is an open payment protocol that standardizes the HTTP 402 Payment Required status code so that API services can be charged via cryptocurrency.
    24|
    25|### Core advantages
    26|
    27||characteristic|illustrate|
    28||------|------|
    29||**No registration required**|You only need your wallet address to pay|
    30||**Instant payment**|It will take effect immediately after being confirmed on the chain.|
    31||**Transparent Pricing**|The payment amount is clearly displayed before the transaction|
    32||**Multi-chain support**|Supports EVM (Base mainnet/testnet) and SVM (Solana mainnet/testnet)|
    33|
    34|### Supported networks
    35|
    36||network|Network ID|Chain ID|illustrate|
    37||------|------------|----------|------|
    38||Base Mainnet|`eip155:8453`|8453|EVM mainnet, **recommended**|
    39||Base Sepolia|`eip155:84532`|84532|EVM testnet|
    40||Solana Mainnet|`solana:mainnet`|-|SVM mainnet|
    41||Solana Devnet|`solana:devnet`|-|SVM testnet|

    60|### The simplest example (fixed amount recharge)
    61|
    62|Automate the entire payment process using the x402 SDK:
    63|
    64|```javascript
    65|import { wrapFetchWithPayment, x402Client } from '@x402/fetch';
    66|import { ExactEvmScheme } from '@x402/evm/exact/client';
    67|import { createWalletClient, custom } from 'viem';
    68|
    69|// 1. Connect wallet
    70|const walletClient = createWalletClient({
    71|    transport: custom(window.ethereum)
    72|});
    73|const [address] = await walletClient.requestAddresses();
    74|
    75|// 2. Create a signer
    76|const signer = {
    77|    address,
    78|    signTypedData: async (params) => walletClient.signTypedData({
    79|        account: address,
    80|        ...params
    81|    })
    82|};
    83|
    84|// 3. Initialize x402 client
    85|const client = new x402Client()
    86|    .register('eip155:*', new ExactEvmScheme(signer));
    87|
    88|// 4. Wrap fetch and initiate a request
    89|const paidFetch = wrapFetchWithPayment(fetch, client);
    90|const response = await paidFetch('https://t.ckcloudai.com/x402/v1/top-up', {
    91|    method: 'POST'
    92|});
    93|
    94|const { Amount, Balance } = await response.json();
    95|console.log(`Recharge successful! Amount: $${Amount}, Balance: $${Balance}`);
    96|```

   100|## core concepts
   101|
   102|### HTTP 402 payment process
   103|
   104|```
   105|┌─────────────────────────────────────────────────────────────────────┐
   106|│ x402 payment process │
   107|├─────────────────────────────────────────────────────────────────────┤
   108|│                                                                     │
   109|│ Client Server │
   110|│    │                                  │                             │
   111|│ │ 1. Initiate a request (no payment header) │ │
   112|│    │ ───────────────────────────────> │                             │
   113|│    │                                  │                             │
   114|│ │ 2. Return 402 + PAYMENT-REQUIRED │ │
   115|│    │ <─────────────────────────────── │                             │
   116|│ │ (Including: amount, payment address, network, etc.) │ │
   117|│    │                                  │                             │
   118|│ │ 3. Create payment payload and sign │ │
   119|│ │ encoded as PAYMENT-SIGNATURE │ │
   120|│    │                                  │                             │
   121|│ │ 4. Re-initiate the request (with payment header) │ │
   122|│    │ ───────────────────────────────> │                             │
   123|│    │                                  │                             │
   124|│ │ 5. Verify signature and settle │
   125|│    │                                  │                             │
   126|│ │ 6. Return 200 + business response │ │
   127|│    │ <─────────────────────────────── │                             │
   128|│    │                                  │                             │
   129|│    └────────────────────────────────────────────────────────────────┘
   130|```
   131|
   132|### Key data structures
   133|
   134|#### PaymentRequired (server-side 402 response)
   135|
   136|```typescript
   137|interface PaymentRequired {
   138|    x402Version: number;           // Protocol version, usually 2
   139|    resource: { url: string };     // Requested resource URL
   140|    accepts: PaymentRequirements[]; // Acceptable payment options
   141|    extensions?: Record<string, unknown>;
   142|}
   143|
   144|interface PaymentRequirements {
   145|    scheme: string;      // Payment plan, such as "exact"
   146|    network: string;     // Network, such as "eip155:8453"
   147|    amount: string;      // Amount (µUSD), such as "1000000" = $1.00
   148|    asset: string;       // USDC contract address
   149|    payTo: string;       // Payment address
   150|    maxTimeoutSeconds: number;
   151|    extra: Record<string, unknown>;
   152|}
   153|```
   154|
   155|#### PaymentPayload (client payment signature)
   156|
   157|```typescript
   158|interface PaymentPayload {
   159|    x402Version: number;
   160|    resource: { url: string };
   161|    accepted: PaymentRequirements;  // Selected payment option
   162|    payload: Record<string, unknown>; // Signed transaction data
   163|    extensions?: Record<string, unknown>;
   164|}
   165|```
   166|
   167|### Amount unit description
   168|
   169||unit|value|illustrate|
   170||------|-----|------|
   171||Dollar|$1.00|User perspective|
   172||Micro Dollar (µUSD)|1,000,000|API parameter `amount`|
   173||USDC smallest unit|1,000,000|On-chain transfer (6 decimal places)|
   174|
   175|**Conversion example**:
   176|- Deposit $1.00 → `amount: "1000000"`
   177|- Deposit $5.00 → `amount: "5000000"`
   178|- Deposit $10.00 → `amount: "10000000"`

   182|## Customized recharge amount
   183|
   184|CKCloud recharge interface supports **customized recharge amount**. This is achieved through x402's **Policy** mechanism.
   185|
   186|### Method 1: Use registerPolicy (recommended)
   187|
   188|`registerPolicy` allows modifying PaymentRequirements before payment creation `PaymentRequirements`，including amount：
   189|
   190|```javascript
   191|import { x402Client } from '@x402/core/client';
   192|import { ExactEvmScheme } from '@x402/evm/exact/client';
   193|
   194|// Custom recharge amount (USD)
   195|const topUpAmountUsd = 10.00;
   196|const topUpAmountMicroUsd = Math.floor(topUpAmountUsd * 1_000_000).toString();
   197|
   198|const client = new x402Client()
   199|    // Registration Strategy: Modify Payment Amount
   200|    .registerPolicy((version, requirements) => {
   201|        return requirements.map(req => ({
   202|            ...req,
   203|            amount: topUpAmountMicroUsd  // Custom amount
   204|        }));
   205|    })
   206|    .register('eip155:*', new ExactEvmScheme(signer));
   207|```
   208|
   209|### Method 2: Manually build payment payload
   210|
   211|Full control over the payment process, suitable for scenarios requiring complex customization:
   212|
   213|```javascript
   214|import { x402Client } from '@x402/core/client';
   215|import { ExactEvmScheme } from '@x402/evm/exact/client';
   216|import { 
   217|    decodePaymentRequiredHeader,
   218|    encodePaymentSignatureHeader 
   219|} from '@x402/core/http';
   220|
   221|async function topUpWithCustomAmount(amountUsd) {
   222|    const amountMicroUsd = Math.floor(amountUsd * 1_000_000).toString();
   223|    
   224|    // 1. Initiate a request and get a 402 response
   225|    let response = await fetch('https://t.ckcloudai.com/x402/v1/top-up', {
   226|        method: 'POST'
   227|    });
   228|    
   229|    if (response.status !== 402) {
   230|        throw new Error(`Expected 402, got ${response.status}`);
   231|    }
   232|    
   233|    // 2. Parse the Payment-Required header
   234|    const paymentRequiredHeader = response.headers.get('PAYMENT-REQUIRED');
   235|    const paymentRequired = decodePaymentRequiredHeader(paymentRequiredHeader);
   236|    
   237|    // 3. Modify the amount
   238|    paymentRequired.accepts = paymentRequired.accepts.map(req => ({
   239|        ...req,
   240|        amount: amountMicroUsd  // Custom amount
   241|    }));
   242|    
   243|    // 4. Create payment payload
   244|    const client = new x402Client()
   245|        .register('eip155:*', new ExactEvmScheme(signer));
   246|    
   247|    const paymentPayload = await client.createPaymentPayload(paymentRequired);
   248|    
   249|    // 5. Encode and send the request
   250|    const paymentSignature = encodePaymentSignatureHeader(paymentPayload);
   251|    
   252|    response = await fetch('https://t.ckcloudai.com/x402/v1/top-up', {
   253|        method: 'POST',
   254|        headers: {
   255|            'PAYMENT-SIGNATURE': paymentSignature
   256|        }
   257|    });
   258|    
   259|    return response.json();
   260|}
   261|
   262|// Usage example
   263|const result = await topUpWithCustomAmount(10.00);
   264|console.log(`Recharge successful! Balance: $${result.Balance}`);
   265|```

   267|### Method 3: Use life cycle hooks
   268|
   269|Use hooks for logging, monitoring, or custom validation:
   270|
   271|```javascript
   272|const client = new x402Client()
   273|    .register('eip155:*', new ExactEvmScheme(signer))
   274|    
   275|    // Before payment is created
   276|    .onBeforePaymentCreation(async (context) => {
   277|        console.log('Payment will be created soon:', {
   278|            network: context.selectedRequirements.network,
   279|            amount: context.selectedRequirements.amount
   280|        });
   281|        
   282|        // Payment can be suspended
   283|        // return { abort: true, reason: 'Amount exceeds limit' };
   284|    })
   285|    
   286|    // After payment is created
   287|    .onAfterPaymentCreation(async (context) => {
   288|        console.log('Payment payload has been created:', context.paymentPayload);
   289|        // Can log to database or send monitoring data
   290|    })
   291|    
   292|    // When payment creation fails
   293|    .onPaymentCreationFailure(async (context) => {
   294|        console.error('Payment creation failed:', context.error);
   295|        // You can try to restore
   296|        // return { recovered: true, payload: alternativePayload };
   297|    });
   298|```

   300|### Method 4: Hard-coded configuration (zero additional requests) ⭐ Recommended
   301|
   302|If the server provides the following configuration information, the front end can skip the first 402 request and build the payment payload directly:
   303|
   304|```javascript
   305|import { x402Client } from '@x402/core/client';
   306|import { ExactEvmScheme } from '@x402/evm/exact/client';
   307|import { encodePaymentSignatureHeader } from '@x402/core/http';
   308|
   309|// ==================== Hardcoded configuration (provided by backend) ====================
   310|const HARDCODED_CONFIG = {
   311|    // Base mainnet configuration
   312|    evm: {
   313|        network: 'eip155:8453',
   314|        payTo: '0x...',           // Payment address (provided by the backend)
   315|        asset: '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913', // Base USDC
   316|        // extra is used for EIP-712 signature and must contain the name and version of USDC
   317|        extra: {
   318|            name: 'USD Coin',
   319|            version: '2',
   320|            decimals: 6,
   321|        }
   322|    },
   323|    // Solana mainnet configuration
   324|    svm: {
   325|        network: 'solana:mainnet',
   326|        payTo: '...',             // Payment address (provided by the backend)
   327|        asset: 'EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v', // Solana USDC
   328|        extra: {}
   329|    }
   330|};
   331|
   332|const BASE_URL = 'https://t.ckcloudai.com';
   333|// ================================================================
   334|
   335|async function topUpWithHardcodedConfig(amountUsd, useSolana = false) {
   336|    const amountMicroUsd = Math.floor(amountUsd * 1_000_000).toString();
   337|    const config = useSolana ? HARDCODED_CONFIG.svm : HARDCODED_CONFIG.evm;
   338|    
   339|    // 1. Build PaymentRequired directly (no need to request backend)
   340|    const paymentRequired = {
   341|        x402Version: 2,
   342|        resource: { url: `${BASE_URL}/x402/v1/top-up` },
   343|        accepts: [{
   344|            scheme: 'exact',
   345|            network: config.network,
   346|            amount: amountMicroUsd,
   347|            asset: config.asset,
   348|            payTo: config.payTo,
   349|            maxTimeoutSeconds: 3600,
   350|            extra: config.extra
   351|        }]
   352|    };
   353|    
   354|    // 2. Create payment payload
   355|    const client = new x402Client()
   356|        .register('eip155:*', new ExactEvmScheme(signer));  // or ExactSvmScheme
   357|    
   358|    const paymentPayload = await client.createPaymentPayload(paymentRequired);
   359|    
   360|    // 3. Send a request (only once!)
   361|    const response = await fetch(`${BASE_URL}/x402/v1/top-up`, {
   362|        method: 'POST',
   363|        headers: {
   364|            'PAYMENT-SIGNATURE': encodePaymentSignatureHeader(paymentPayload)
   365|        }
   366|    });
   367|    
   368|    return response.json();
   369|}
   370|
   371|// Usage example
   372|const result = await topUpWithHardcodedConfig(10.00);
   373|console.log(`Recharge successful! Balance: $${result.Balance}`);
   374|```

   400|## API interface details
   401|
   402|### Recharge interface
   403|
   404||project|value|
   405||-----|-----|
   406|| **URL** | `/x402/v1/top-up` |
   407|| **Method** | `POST` |
   408||**Certification**|x402 payment|
   409|
   410|### Request header
   411|
   412||Parameter name|type|Required|illustrate|
   413||-------|------|------|------|
   414||`PAYMENT-SIGNATURE`|string|yes|Base64 encoded payment payload|
   415|
   416|### response parameters
   417|
   418|**Successful response (HTTP 200)**:
   419|
   420|```json
   421|{
   422|    "Pubkey": "0x774b3f6C5a6F8e2D9A1B3C4d5E6f7A8b9C0D1e2F",
   423|    "Amount": 10.0,
   424|    "Balance": 25.5
   425|}
   426|```
   427|
   428||Parameter name|type|illustrate|
   429||-------|------|------|
   430||`Pubkey`|string|User wallet address|
   431||`Amount`|float|Amount of this recharge (USD)|
   432||`Balance`|float|Account balance after recharge (USD)|
   433|
   434|**Error response**:
   435|
   436||HTTP status code|illustrate|
   437||------------|------|
   438||400|Payment signature format is incorrect or verification failed|
   439||402|Payment required (missing PAYMENT-SIGNATURE header)|
   440||500|Payment processing failed|
