# CKCloud x402 Recharge API Documentation

This document describes how to recharge a CKCloud account using the **x402 protocol**. x402 is a payment protocol based on the HTTP 402 status code that allows users to directly fund their accounts through cryptocurrency (USDC).

## Overview

### What is x402?

**x402** is an open payment protocol that standardizes the HTTP 402 Payment Required status code so that API services can be charged via cryptocurrency.

### Core advantages

|characteristic|illustrate|
|------|------|
|**No registration required**|You only need your wallet address to pay|
|**Instant payment**|It will take effect immediately after being confirmed on the chain.|
|**Transparent Pricing**|The payment amount is clearly displayed before the transaction|
|**Multi-chain support**|Supports EVM (Base mainnet/testnet) and SVM (Solana mainnet/testnet)|

### Supported networks

|network|Network ID|Chain ID|illustrate|
|------|------------|----------|------|
|Base Mainnet|`eip155:8453`|8453|EVM mainnet, **recommended**|
|Base Sepolia|`eip155:84532`|84532|EVM testnet|
|Solana Mainnet|`solana:mainnet`|-|SVM mainnet|
|Solana Devnet|`solana:devnet`|-|SVM testnet|

## quick start

### Install dependencies

```bash
# Use npm
npm install @x402/fetch @x402/core @x402/evm viem

# Solana support (optional)
npm install @x402/svm @solana/kit @scure/base

# or use yarn
yarn add @x402/fetch @x402/core @x402/evm viem
```

### The simplest example (fixed amount recharge)

Automate the entire payment process using the x402 SDK:

```javascript
import { wrapFetchWithPayment, x402Client } from '@x402/fetch';
import { ExactEvmScheme } from '@x402/evm/exact/client';
import { createWalletClient, custom } from 'viem';

// 1. Connect wallet
const walletClient = createWalletClient({
    transport: custom(window.ethereum)
});
const [address] = await walletClient.requestAddresses();

// 2. Create a signer
const signer = {
    address,
    signTypedData: async (params) => walletClient.signTypedData({
        account: address,
        ...params
    })
};

// 3. Initialize x402 client
const client = new x402Client()
    .register('eip155:*', new ExactEvmScheme(signer));

// 4. Wrap fetch and initiate a request
const paidFetch = wrapFetchWithPayment(fetch, client);
const response = await paidFetch('https://t.ckcloudai.com/x402/v1/top-up', {
    method: 'POST'
});

const { Amount, Balance } = await response.json();
console.log(`Recharge successful! Amount: $${Amount}, Balance: $${Balance}`);
```

## core concepts

### HTTP 402 payment process

```
┌─────────────────────────────────────────────────────────────────────┐
│ x402 payment process │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│ Client Server │
│    │                                  │                             │
│ │ 1. Initiate a request (no payment header) │ │
│    │ ───────────────────────────────> │                             │
│    │                                  │                             │
│ │ 2. Return 402 + PAYMENT-REQUIRED │ │
│    │ <─────────────────────────────── │                             │
│ │ (Including: amount, payment address, network, etc.) │ │
│    │                                  │                             │
│ │ 3. Create payment payload and sign │ │
│ │ encoded as PAYMENT-SIGNATURE │ │
│    │                                  │                             │
│ │ 4. Re-initiate the request (with payment header) │ │
│    │ ───────────────────────────────> │                             │
│    │                                  │                             │
│ │ 5. Verify signature and settle │
│    │                                  │                             │
│ │ 6. Return 200 + business response │ │
│    │ <─────────────────────────────── │                             │
│    │                                  │                             │
│    └────────────────────────────────────────────────────────────────┘
```

### Key data structures

#### PaymentRequired (server-side 402 response)

```typescript
interface PaymentRequired {
    x402Version: number;           // Protocol version, usually 2
    resource: { url: string };     // Requested resource URL
    accepts: PaymentRequirements[]; // Acceptable payment options
    extensions?: Record<string, unknown>;
}

interface PaymentRequirements {
    scheme: string;      // Payment plan, such as "exact"
    network: string;     // Network, such as "eip155:8453"
    amount: string;      // Amount (µUSD), such as "1000000" = $1.00
    asset: string;       // USDC contract address
    payTo: string;       // Payment address
    maxTimeoutSeconds: number;
    extra: Record<string, unknown>;
}
```

#### PaymentPayload (client payment signature)

```typescript
interface PaymentPayload {
    x402Version: number;
    resource: { url: string };
    accepted: PaymentRequirements;  // Selected payment option
    payload: Record<string, unknown>; // Signed transaction data
    extensions?: Record<string, unknown>;
}
```

### Amount unit description

|unit|value|illustrate|
|------|-----|------|
|Dollar|$1.00|User perspective|
|Micro Dollar (µUSD)|1,000,000|API parameter `amount`|
|USDC smallest unit|1,000,000|On-chain transfer (6 decimal places)|

**Conversion example**:
- Deposit $1.00 → `amount: "1000000"`
- Deposit $5.00 → `amount: "5000000"`
- Deposit $10.00 → `amount: "10000000"`

## Customized recharge amount

CKCloud recharge interface supports **customized recharge amount**. This is achieved through x402's **Policy** mechanism.

### Method 1: Use registerPolicy (recommended)

`registerPolicy` allows modifying PaymentRequirements before payment creation `PaymentRequirements`，including amount：

```javascript
import { x402Client } from '@x402/core/client';
import { ExactEvmScheme } from '@x402/evm/exact/client';

// Custom recharge amount (USD)
const topUpAmountUsd = 10.00;
const topUpAmountMicroUsd = Math.floor(topUpAmountUsd * 1_000_000).toString();

const client = new x402Client()
    // Registration Strategy: Modify Payment Amount
    .registerPolicy((version, requirements) => {
        return requirements.map(req => ({
            ...req,
            amount: topUpAmountMicroUsd  // Custom amount
        }));
    })
    .register('eip155:*', new ExactEvmScheme(signer));
```

### Method 2: Manually build payment payload

Full control over the payment process, suitable for scenarios requiring complex customization:

```javascript
import { x402Client } from '@x402/core/client';
import { ExactEvmScheme } from '@x402/evm/exact/client';
import { 
    decodePaymentRequiredHeader,
    encodePaymentSignatureHeader 
} from '@x402/core/http';

async function topUpWithCustomAmount(amountUsd) {
    const amountMicroUsd = Math.floor(amountUsd * 1_000_000).toString();
    
    // 1. Initiate a request and get a 402 response
    let response = await fetch('https://t.ckcloudai.com/x402/v1/top-up', {
        method: 'POST'
    });
    
    if (response.status !== 402) {
        throw new Error(`Expected 402, got ${response.status}`);
    }
    
    // 2. Parse the Payment-Required header
    const paymentRequiredHeader = response.headers.get('PAYMENT-REQUIRED');
    const paymentRequired = decodePaymentRequiredHeader(paymentRequiredHeader);
    
    // 3. Modify the amount
    paymentRequired.accepts = paymentRequired.accepts.map(req => ({
        ...req,
        amount: amountMicroUsd  // Custom amount
    }));
    
    // 4. Create payment payload
    const client = new x402Client()
        .register('eip155:*', new ExactEvmScheme(signer));
    
    const paymentPayload = await client.createPaymentPayload(paymentRequired);
    
    // 5. Encode and send the request
    const paymentSignature = encodePaymentSignatureHeader(paymentPayload);
    
    response = await fetch('https://t.ckcloudai.com/x402/v1/top-up', {
        method: 'POST',
        headers: {
            'PAYMENT-SIGNATURE': paymentSignature
        }
    });
    
    return response.json();
}

// Usage example
const result = await topUpWithCustomAmount(10.00);
console.log(`Recharge successful! Balance: $${result.Balance}`);
```

### Method 3: Use life cycle hooks

Use hooks for logging, monitoring, or custom validation:

```javascript
const client = new x402Client()
    .register('eip155:*', new ExactEvmScheme(signer))
    
    // Before payment is created
    .onBeforePaymentCreation(async (context) => {
        console.log('Payment will be created soon:', {
            network: context.selectedRequirements.network,
            amount: context.selectedRequirements.amount
        });
        
        // Payment can be suspended
        // return { abort: true, reason: 'Amount exceeds limit' };
    })
    
    // After payment is created
    .onAfterPaymentCreation(async (context) => {
        console.log('Payment payload has been created:', context.paymentPayload);
        // Can log to database or send monitoring data
    })
    
    // When payment creation fails
    .onPaymentCreationFailure(async (context) => {
        console.error('Payment creation failed:', context.error);
        // You can try to restore
        // return { recovered: true, payload: alternativePayload };
    });
```

### Method 4: Hard-coded configuration (zero additional requests) ⭐ Recommended

If the server provides the following configuration information, the front end can skip the first 402 request and build the payment payload directly:

```javascript
import { x402Client } from '@x402/core/client';
import { ExactEvmScheme } from '@x402/evm/exact/client';
import { encodePaymentSignatureHeader } from '@x402/core/http';

// ==================== Hardcoded configuration (provided by backend) ====================
const HARDCODED_CONFIG = {
    // Base mainnet configuration
    evm: {
        network: 'eip155:8453',
        payTo: '0x...',           // Payment address (provided by the backend)
        asset: '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913', // Base USDC
        // extra is used for EIP-712 signature and must contain the name and version of USDC
        extra: {
            name: 'USD Coin',
            version: '2',
            decimals: 6,
        }
    },
    // Solana mainnet configuration
    svm: {
        network: 'solana:mainnet',
        payTo: '...',             // Payment address (provided by the backend)
        asset: 'EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v', // Solana USDC
        extra: {}
    }
};

const BASE_URL = 'https://t.ckcloudai.com';
// ================================================================

async function topUpWithHardcodedConfig(amountUsd, useSolana = false) {
    const amountMicroUsd = Math.floor(amountUsd * 1_000_000).toString();
    const config = useSolana ? HARDCODED_CONFIG.svm : HARDCODED_CONFIG.evm;
    
    // 1. Build PaymentRequired directly (no need to request backend)
    const paymentRequired = {
        x402Version: 2,
        resource: { url: `${BASE_URL}/x402/v1/top-up` },
        accepts: [{
            scheme: 'exact',
            network: config.network,
            amount: amountMicroUsd,
            asset: config.asset,
            payTo: config.payTo,
            maxTimeoutSeconds: 3600,
            extra: config.extra
        }]
    };
    
    // 2. Create payment payload
    const client = new x402Client()
        .register('eip155:*', new ExactEvmScheme(signer));  // or ExactSvmScheme
    
    const paymentPayload = await client.createPaymentPayload(paymentRequired);
    
    // 3. Send a request (only once!)
    const response = await fetch(`${BASE_URL}/x402/v1/top-up`, {
        method: 'POST',
        headers: {
            'PAYMENT-SIGNATURE': encodePaymentSignatureHeader(paymentPayload)
        }
    });
    
    return response.json();
}

// Usage example
const result = await topUpWithHardcodedConfig(10.00);
console.log(`Recharge successful! Balance: $${result.Balance}`);
```

## API interface details

### Recharge interface

|project|value|
|-----|-----|
| **URL** | `/x402/v1/top-up` |
| **Method** | `POST` |
|**Certification**|x402 payment|

### Request header

|Parameter name|type|Required|illustrate|
|-------|------|------|------|
|`PAYMENT-SIGNATURE`|string|yes|Base64 encoded payment payload|

### response parameters

**Successful response (HTTP 200)**:

```json
{
    "Pubkey": "0x774b3f6C5a6F8e2D9A1B3C4d5E6f7A8b9C0D1e2F",
    "Amount": 10.0,
    "Balance": 25.5
}
```

|Parameter name|type|illustrate|
|-------|------|------|
|`Pubkey`|string|User wallet address|
|`Amount`|float|Amount of this recharge (USD)|
|`Balance`|float|Account balance after recharge (USD)|

**Error response**:

|HTTP status code|illustrate|
|------------|------|
|400|Payment signature format is incorrect or verification failed|
|402|Payment required (missing PAYMENT-SIGNATURE header)|
|500|Payment processing failed|
